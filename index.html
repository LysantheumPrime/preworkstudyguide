NodeJS

  • 1 way to use node is in REPL mode which stands for Read Print Loop
  • When you type node into the command line it allows you to run Javascript code and will print out the results, For example we can type console.log('Hello World') and it will log in the console.
  • CTRL+C closes the console
  • Create a NodeJS file in VS-Code by typing index.js
  • NodeJS has a handful of built-in identifiers, one of those globals is console which logs out values to the terminal
  • You can run commands in the command line by using the path to the file such as "node index.js", but if it's just a index file you can just do "node ." as it is the parent directory
  • There is another indentifier called global this object is a namespace that is available throughout the entire node process for example if we assigned a lucky number property to global we could then access it anywhere else in our code.
  • Example: console.log(global.luckyNum); global.luckyNum = '23'; console.log(global.luckyNum); It will print the lucky number into the console.
  • Important global to be familiar with is "process" which gives you access to the currently running node you might use it to check the current platform or operating system you're currently on. console.log(process.platform);
  • Before a node process finishes it emits an event named "exit"
  • Example: process.on('exit', function() {})
  • The function isn't called the first time node.js sees it it's only called after the exit event occurs at some point in the future.
  • Example of a "created from scratch" or CFS const { EventEmitter } = require('events'); const eventEmitter = new EventEmitter(); eventEmitter.on('lunch', () => { console.log('yum') }) eventEmitter.emit('lunch');
  • Node has a built-in file system module called fs it can read write and delete files on the file system among other things and it can also do things in a blocking and non-blocking way.
  • Anytime you see a function end in sync === blocking in other words it needs to finish all of its work before any of your other code can run
